Concept / Pre-Compliance Review
Massachusetts MassPAT

Two Architecture Paths

Both approaches detect outside opioid prescriptions in surgical patients at the 7-day post-op mark. They differ in technical integration requirements, legal complexity, and time to deploy. Massachusetts uses MassPAT (PMP AWARxE platform via Bamboo Health), with mandatory query requirements under MGL Ch. 94C Sec. 24A for Schedule II/III opioids and benzodiazepines. Lahey Clinic is part of Beth Israel Lahey Health, which runs Epic.

Compliance Prerequisites (Both Paths)

  • Massachusetts law (MGL Ch. 94C Sec. 24A) mandates PDMP query before prescribing Schedule II or III opioids or benzodiazepines. Authorized queriers include the prescribing provider and their registered delegates. Automated/batch queries require approval from the MA Department of Public Health Drug Control Program.
  • HIPAA minimum-necessary standard applies. Query results must be stored in or linked to the patient's medical record, not in a standalone database.
  • Delegate authority must be established in writing and registered in MassPAT. Automated systems acting as a delegate require the same credentialing as a human delegate under MA DPH policy.
  • MassPAT is administered via PMP AWARxE (Bamboo Health). Integration with Epic at Beth Israel Lahey Health requires a Bamboo Health PMP Gateway agreement and institutional IT approval.

Path B: Standalone Tool

Surgeon or coordinator uploads patient list. Tool queries state PDMP API directly and returns flagged results outside the EHR

Standalone Build
01
Patient List Upload
Coordinator exports surgical schedule from Epic (CSV: MRN, name, DOB, procedure date, surgeon NPI). Uploads to standalone PDMP tool. Data must stay within HIPAA-compliant infrastructure.
BAA with tool vendor required
02
Day-7 Filter Applied
Tool calculates current_date minus procedure_date. Patients at day 7 (configurable: 6-8 day window) are queued for PDMP query. Future patients queued for future runs. Historical patients archived.
Configurable window
03
MassPAT API Batch Query
Tool submits queries to MassPAT via Bamboo Health PMP Gateway API using surgeon NPI as the requesting provider. Requires MA DPH Drug Control Program approval for automated/batch access plus a Bamboo Health vendor agreement. Returns Schedule II-V prescription history per patient for configurable lookback window (default: 30 days post-op).
MA DPH approval required Bamboo Health agreement required
04
Flag Logic Applied
Tool flags: (1) any opioid Rx from a different prescriber, (2) Rx for same drug class as post-op prescription, (3) Rx within 48h of discharge, (4) controlled substance from 3+ unique prescribers in 30 days. Clean patients marked green.
Configurable rules
05
Dashboard + Alert Routing
Flagged patients surfaced in surgeon dashboard. Alert can route to coordinator InBasket or secure message. Results must be copied to Epic chart to maintain documentation continuity. This step requires manual action or an Epic API write-back.
Manual chart documentation Epic write-back: optional Phase 2

RECOMMENDATION: For Lahey Clinic / Beth Israel Lahey Health, Path A via Epic BPA is the correct long-term architecture. The system-wide Bamboo Health PMP Gateway integration is already in place, documentation is automatic, and mandatory MassPAT query is already a compliance requirement under MGL Ch. 94C Sec. 24A. The incremental build needed is a BPA rule that fires at post-op day 7 -- an Epic governance request, not a new vendor relationship. Path B (standalone tool) is viable for an independent or single-surgeon context outside the Lahey system where Epic integration is not available.

Massachusetts MassPAT API Guide

Massachusetts uses MassPAT, built on Bamboo Health's PMP AWARxE platform. API access for automated queries is available via the Bamboo Health PMP Gateway. MGL Ch. 94C Sec. 24A mandates PDMP query before prescribing Schedule II or III opioids or benzodiazepines, making this the only state PDMP tool in the country with a statutory prescribing mandate tied to it. Interstate data sharing covers all New England states plus New York via PMP InterConnect.

Massachusetts (Primary)

Mandate: MGL Ch. 94C Sec. 24A -- required before Sch II/III opioids and benzodiazepines

Massachusetts MassPAT: API Authorization Steps for Dr. Specht / Lahey Clinic

  • Register or confirm existing MassPAT account at massachusetts.pmpaware.net. The surgeon's DEA number and MA controlled substance registration must be active and on file. Contact mapmp.dph@mass.gov or 617-753-7310 to initiate institutional API access discussion.
  • Submit a request to MA DPH Drug Control Program for batch/automated query authorization, identifying the requesting provider (surgeon NPI/DEA), the institutional entity (Beth Israel Lahey Health / Lahey Clinic), and the automated system requesting delegate access.
  • Execute a Data Use Agreement with MA DPH specifying permitted uses (post-op opioid stewardship monitoring), data retention policy, and breach notification obligations under MGL Ch. 93H and HIPAA.
  • Contact Bamboo Health (PMP AWARxE support: 855-562-4767) to initiate PMP Gateway vendor agreement. If Beth Israel Lahey Health already has an institutional PMP Gateway contract for Epic integration, a separate agreement for the standalone tool may require an addendum rather than a new contract.
  • Establish a HIPAA Business Associate Agreement with the tool developer before any PHI transmission occurs. Document delegate authorization for automated querying in compliance records, signed by Dr. Specht as the supervising prescriber.
  • Note: Because MGL Ch. 94C Sec. 24A already mandates MassPAT query before Sch II/III opioid prescribing, the compliance argument for this stewardship tool is straightforward. The automated batch query extends mandatory behavior into a proactive post-op monitoring workflow rather than creating a new obligation.
